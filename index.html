A Single Page JavaScript Front-end Framework

Why You Should Use KissJS?

When developing web applications, the language or framework being used should help the developer create applications fast and efficiently, but at the same time it should be simple enough for them to grasp it quickly. This proposed framework takes into consideration the pros and cons of modern Single Page Application (SPA) development frameworks and tries to use some their approaches to achieve efficiency, and at the same time use its own ways to simplify other tasks that are complex in the aforementioned frameworks. The main barrier this framework is trying to overcome is designing a method for the state management of the application to be simple enough for new developers to learn and for anyone to get the grasp of quickly. In addition to that, this research will also focus on the approaches behind other aspects of the framework, such as how the framework handles change detection, the syntax of the framework and the development of an extension for various code editors that will aid developers when developing with this framework.

  • 01
  • In-built State Management A state management system that comes built into the framework, that is also easy to learn and develop with.
  • 02
  • Framework Syntax A simple syntax to be used within the framework along with a preset to trans-compile this syntax to valid JavaScript.
  • 03
  • Change Detection Mechanism A change detection mechanism utilizing the virtual DOM and a diff algorithm to handle UI changes
  • 04
  • KissJS langugage support for VS code editor. Programmatic and Declarative language features with built in snippet completion support for the KissJS new syntaxes. NPM package for KissJS error handling/linting.

Literature Survey

    In order to begin the development of the framework various documentation and research were looked into in order to get a better understanding of how all the components in a SPA framework come together in order to create applications.
    When taking into consideration the syntax of the framework, the syntax cannot simply be read and compiled by the browser. The syntax must first be converted into a valid JavaScript object. This can be done using a trans-compiler, and a preset based on the babel core must be created to accomplish this [1].
    Regarding the change detection mechanism of the framework, there are various methods available. The most commonly used ones are Key-Value Observation, Dirty Checking and Virtual DOM [2]. Each method was found to have its own advantages and disadvantages however, the virtual DOM method proved to be the most effective method overall [3].
    Various researches were referred to for the creation of the language support for the framework, and the most helpful of resources were on Ballerinalang and its Language Server [4]. Using a Language Server Protocol along with the Language Server, the extension for the framework could be used.

Research Gap

  • The main drawback identified from the current frameworks is their lack of a simplistic state management solution. Usually these have to be added as an extra dependency, and are usually quite complex for new developers to use.
  • Most modern frameworks also do not have a direct method to identify static elements in the DOM. Identifying these could help reduce re-rendering times, thereby making the application more efficient.
  • As it is, no language support exists for KissJS. This would mean that developers working of .kjs files would face difficulty. Therefore, there is a need for an extension that would help in the development of KissJS applications.

Research Objectives

  • Main Objective

    • The main objective of this project can be thought of as the development of a frontend framework that comes with an in-built state management mechanism. In addition to this main functionality, the framework itself should be fast and efficient when it comes to detecting changes and should be easy to learn.

  • Sub Objectives

    • In order to achieve the main functionality of the framework, certain sub-objectives can be drawn out in order to fulfill the project’s requirements. These sub-objectives are as follows: • Implementing state management • Creation of the virtual DOM from the trans-compiled JavaScript object • Implementation of a diff algorithm to compute the differences between the previous and new virtual DOMs in order to update the original DOM. • Mechanism to map the virtual DOM nodes to the nodes in the DOM. • Implement unique Language support for the newly proposed framework’s language syntaxes, functions, and classes. All the declarative and programmatic language features should be included to make the modern code editors/IDEs like VS code an efficient, and smooth development environment for the developers. Using LSP, the communication path between client (code editor) and Language Server will be implemented. • Implement an in-build snippets to increase the development accuracy. • Publish the KissJS language extension to the VS code marketplace. • Implement kissjs-eslint-plugin to get unique diagnosis messages for the new language support and publish this to the NPM directory.

Methodology

    The main outcome of this project is to create a new frontend framework for SPA development. This framework would come with an in-built state management system that users can learn and use easily. In addition to this the main components of the framework are, a syntax for the framework, change detection mechanism and a language extension to support development in code editors.
    At a high-level, the way the framework operates is as follows. Once the user has created his components with the KissJS syntax, our custom trans-compiler will convert that into JavaScript objects that can be understood by the browser. These objects are passed into the change detection part of the framework, and here they will be used to create a virtual DOM and then rendered on the screen.
    Once any changes have been made, the virtual DOM will be traversed in order to identify which nodes have undergone change. The duty of the change detection mechanism is to identify these nodes and only re-render them. A diff algorithm used with the virtual DOM will accomplish the task of traversing and identifying changes. When the changes are identified they will be reflected in the DOM.
    And using the KissJS language support, the client (user machine) will be communicating with the KissJS language server in order to aid in development. The extension will provide various functionalities such as snippet completion, bracket highlighting, error highlighting, etc.
